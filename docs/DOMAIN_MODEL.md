# Translator with educational functions - Диаграмма доменной модели

## Обзор
Этот документ описывает ключевые сущности и их связи в области применения приложения Translator, на основе спецификации требований к системе (SRS). Эта модель служит концептуальным фундаментом для схемы базы данных и объектно-ориентированного дизайна системы.

## Описание диаграммы классов
Основной домен Translator вращается вокруг управления переводами, пользовательскими данными и образовательными функциями. Основные сущности:

* **`User`**: Центральный актор. Все переводы и викторины привязаны к конкретному пользователю.
* **`Translation`**: Представляет оригинальный и переведенный тексты и связанную с ними метаданные.
* **`Quiz`**: Набор вопросов, созданных на основе оригинального и переведенного текста.
* **`AuthToken`**: Представляет действительную сессию для аутентифицированного пользователя.

На следующей диаграмме показаны связи между этими сущностями:

### Основные связи объяснены:

1. **User - Translation (Один-ко-многим)**:
   * Один `User` может иметь много сущностей `Translation`. Каждый `Translation` принадлежит ровно одному `User`.

2. **User - Quiz (Один-ко-многим)**:
   * Один `User` может создавать много сущностей `Quiz`. Каждый `Quiz` принадлежит ровно одному `User`.

3. **Translation - Quiz (Один-ко-многим)**:
   * Один `Translation` может быть основой для многочисленных `Quiz`. Каждый `Quiz` основан на одном `Translation`.

4. **User - AuthToken (Один-ко-многим)**:
   * Один `User` может иметь несколько активных сессий `AuthToken` (например, вход на нескольких устройствах). Каждый `AuthToken` принадлежит ровно одному `User`.

### Атрибуты:

* **`User`**: `id` (Primary Key), `username`, `email`, `password_hash`, `created_at`, `last_login`.
* **`Translation`**: `id` (PK), `original_text`, `translated_text`, `source_lang`, `target_lang`, `user_id` (Foreign Key to User), `created_at`.
* **`Quiz`**: `id` (PK), `title`, `original_text`, `translated_text`, `source_lang`, `target_lang`, `user_id` (FK to User), `translation_id` (FK to Translation).
* **`QuizQuestion`**: `id` (PK), `quiz_id` (FK to Quiz), `question_text`, `correct_answer`, `options` (JSON), `question_order`.
* **`AuthToken`**: `id` (PK), `token`, `user_id` (FK to User), `expires_at`.

Эта модель обеспечивает изоляцию данных между пользователями и предоставляет гибкую структуру для управления переводами и образовательными функциями.
