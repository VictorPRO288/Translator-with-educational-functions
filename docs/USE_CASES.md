## Основные варианты использования

### UC-1: Создание перевода
**Актор:** Аутентифицированный пользователь  
**Цель:** Перевести текст с одного языка на другой  
**Предусловия:** Пользователь вошёл в систему  
**Основной поток:**
1. Пользователь вводит текст в поле ввода
2. Выбирает язык оригинала и язык перевода
3. Нажимает кнопку "Перевести"
4. Система отправляет текст во внешний API (Google Gemini)
5. Система получает перевод и отображает его пользователю
6. Перевод сохраняется в истории пользователя

**Альтернативный поток:** Если API недоступно, система показывает сообщение об ошибке

### UC-2: Прохождение викторины
**Актор:** Аутентифицированный пользователь  
**Цель:** Пройти викторину для проверки знания перевода  
**Предусловия:** У пользователя есть сохраненный перевод  
**Основной поток:**
1. Пользователь выбирает перевод для создания викторины
2. Система генерирует вопросы на основе оригинального и переведенного текста
3. Пользователь отвечает на вопросы
4. Система проверяет ответы и предоставляет обратную связь
5. По завершении викторины отображаются результаты

**Альтернативный поток:** Если викторина для этого текста уже существует, система использует существующую викторину

### UC-3: Получение подсказок
**Актор:** Аутентифицированный пользователь  
**Цель:** Получить рекомендации по улучшению текста  
**Предусловия:** Пользователь вошёл в систему  
**Основной поток:**
1. Пользователь вводит текст для анализа
2. Нажимает кнопку получения подсказок
3. Система отправляет текст в LanguageTool API
4. Система получает анализ грамматики и стиля
5. Результаты отображаются пользователю в понятной форме

### UC-4: Просмотр истории переводов
**Актор:** Аутентифицированный пользователь  
**Цель:** Просмотреть предыдущие переводы  
**Предусловия:** Пользователь вошёл в систему и имеет сохраненные переводы  
**Основной поток:**
1. Пользователь переходит на страницу истории переводов
2. Система загружает переводы пользователя из базы данных
3. Переводы отображаются в виде списка
4. Пользователь может выбрать конкретный перевод для просмотра

**Альтернативный поток:** Если переводов нет, система показывает соответствующее сообщение

### UC-5: Регистрация нового пользователя
**Актор:** Неаутентифицированный пользователь  
**Цель:** Создать новый аккаунт в системе  
**Предусловия:** Пользователь находится на странице регистрации  
**Основной поток:**
1. Пользователь заполняет форму регистрации
2. Вводит имя пользователя, email и пароль
3. Нажимает кнопку регистрации
4. Система проверяет уникальность учетных данных
5. При успешной проверке создает новый аккаунт
6. Пользователь перенаправляется на страницу входа

**Альтернативный поток:** Если учетные данные уже существуют, система показывает ошибку
