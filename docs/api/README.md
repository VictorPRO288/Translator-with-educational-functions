# Документация по API

## Обзор

Наше приложение предоставляет REST API для функций перевода и изучения языка. Все API endpoints доступны по базовому URL: `http://localhost:8080/api/`

## Аутентификация

Большинство API endpoints не требуют аутентификации, но для некоторых функций может потребоваться API ключ.

## Основные Endpoints

### 1. Управление переводами

#### GET `/api/translations`
Получить список всех переводов

#### POST `/api/translations`
Создать новый перевод
- Тело запроса: 
  ```json
  {
    "text": "текст для перевода",
    "sourceLang": "исходный язык",
    "targetLang": "язык перевода"
  }
  ```

#### GET `/api/translations/{id}`
Получить перевод по ID

#### GET `/api/translations/by-language?targetLang={language}`
Получить переводы по целевому языку

#### DELETE `/api/translations/{id}`
Удалить перевод по ID

### 2. Викторины

#### POST `/api/quizzes`
Создать викторину по переданному тексту
- Тело запроса:
  ```json
  {
    "originalText": "оригинальный текст",
    "translatedText": "переведенный текст",
    "sourceLang": "исходный язык",
    "targetLang": "язык перевода"
  }
  ```

### 3. Подсказки

#### POST `/api/hints`
Получить подсказки для улучшения текста

### 4. Управление языками

#### GET `/api/translations/languages`
Получить список поддерживаемых языков

## Формат ответа

Успешные ответы возвращаются с кодом 200 и содержат JSON данные.
Ошибки возвращаются с соответствующим HTTP кодом ошибки и описанием в формате:
```json
{
  "timestamp": "дата и время",
  "message": "описание ошибки",
  "details": "дополнительная информация"
}
```
